## 1. 引言

### 1.1 编写目的

本文档明确保险产品中心后台管理系统的功能需求、非功能需求、数据要求、接口要求、验收标准和约束边界，为系统设计、开发、测试、评审和上线验收提供依据。

预期读者包括产品经理、研发工程师、测试工程师、UI/UX 设计师、项目经理、运维人员、合规法务人员、精算人员和业务运营人员。

### 1.2 项目背景

当前保险产品定义散落于多个业务系统，产品配置、审核、发布、上下架依赖人工流转和多系统重复录入，存在效率低、错误率高、版本不可追溯、下游消费口径不一致等问题。

保险产品中心作为保险业务中台能力，聚焦寿险及分红险产品的集中管理，统一承载产品定义、版本管理、险种责任、投保方案、费率、健康告知、前端联动、供应商关联、审核发布和开放查询能力，支撑业务敏捷运营。

### 1.3 术语定义

| 术语 | 定义 |
| --- | --- |
| 产品 | 保险公司面向客户销售的一项保险服务，如“XX 终身寿险（分红型）”。 |
| 险种 | 产品下的具体保障分类，本系统涉及主险、附加险，均为寿险范畴。 |
| 责任 | 保险公司承担的给付或补偿义务，如身故保险金、全残保险金、满期生存金。 |
| 费率表 | 按年龄、性别、职业等级、保险期间、缴费期间、保额等维度计算保费的规则集合。 |
| 条款 | 保险合同相关文本文件，包括保险条款、投保须知、分红说明等。 |
| 产品组合 | 将多个独立寿险产品打包销售的整体方案。 |
| 投保要素 | 投保过程中需要客户填写的字段集合，如姓名、证件号、职业、受益人等。 |
| 健康告知 | 投保时用于评估被保险人健康状况的问卷。 |
| 投保区间 | 客户可选的保单参数范围，如保险期间、缴费期间、保额档位。 |
| 供应商 | 与产品相关的再保公司、第三方服务商、渠道商等。 |
| 前端联动 | 后台配置规则驱动前端页面动态显隐、校验、互斥、试算触发等行为。 |
| 分红险 | 保险公司将实际经营成果优于定价假设的盈余，按一定比例向保单持有人分配的寿险产品。 |
| 分红方式 | 现金分红、增额分红、累积生息、混合方式等。 |
| 发布快照 | 产品版本发布时生成的不可变配置包，供开放接口和下游系统读取。 |

### 1.4 系统范围边界

本系统定位为寿险产品中心，仅支持以下产品的全流程配置管理：

- 传统寿险：定期寿险、终身寿险、两全保险。
- 分红险：分红型终身寿险、分红型两全保险。

本系统不覆盖以下范围：

- 不支持健康险、意外险、财产险、车险、责任险等非寿险产品。
- 不直接负责保单承保、支付、核保决策、理赔、佣金结算。
- 不在产品中心内实现复杂规则引擎；复杂核保、理赔、精算规则通过规则编码、规则服务或精算引擎引用。
- 不存储条款附件的大容量二进制文件，仅保存外部文件 ID、文件版本、预览地址和关联关系。
- 不执行真实红利分配计算，红利演示参数由精算团队提供，真实计算由核心系统或精算引擎完成。

### 1.5 适用范围

系统面向保险公司总部产品部、精算部、运营部、合规法务部、核保运营部、供应商管理部等内部人员，管理寿险及分红险产品从草稿、审核、发布、上架、下架到历史追溯的完整生命周期。

## 2. 总体描述

### 2.1 用户角色与权限

系统采用 RBAC 权限模型，支持按钮级权限和数据权限控制。

| 角色 | 权限描述 |
| --- | --- |
| 产品配置员 | 创建、编辑产品草稿，配置基础信息、险种责任、投保要素、投保方案、条款、分红参数，提交审核。 |
| 精算师 | 上传费率表、维护费率因子、审核费率配置，维护分红型产品的红利演示参数。 |
| 法务/合规审核员 | 审核条款完整性、产品合规性、分红说明和销售材料。 |
| 核保/运营岗 | 审核健康告知、职业规则、投保要素、投保规则。 |
| 产品审批人 | 最终审批产品发布，执行上架、下架、定时上下架。 |
| 供应商管理员 | 维护供应商档案、合同、证照、产品关联、费率和结算配置。 |
| 系统管理员 | 管理用户、角色、菜单、按钮权限、基础字典、系统参数。 |
| 只读观察者 | 查看产品全生命周期数据、版本、审核记录、报表，无编辑权限。 |

### 2.2 系统边界与上下游

上游系统：

- 精算引擎：费率因子、费率表、红利演示参数导入或计算结果引用。
- 规则引擎：核保规则、职业限制、复杂业务规则引用。
- 条款文库系统：条款文件存储、版本、预览。
- 统一认证与权限系统：用户、组织、角色、登录态。

下游系统：

- 保险核心系统：产品定义、版本、责任、费率、分红配置消费。
- 销售渠道系统：APP、H5、小程序、中介平台，读取投保页面配置和试算能力。
- 保单管理系统：使用产品版本快照完成保单承保记录追溯。
- 理赔系统：读取责任、条款、产品版本历史信息。
- 报表与数据仓库：消费发布事件、操作日志和产品维度数据。

### 2.3 假设与约束

- 公司已有统一编码规则，产品编码支持自动生成和权限受控的手动输入。
- 产品版本号按小版本递增，例如 `V1.0`、`V1.1`；已发布版本不可原地修改。
- 动态字段扩展、联动规则、验证规则必须配置化，但不能执行任意脚本。
- 后台写接口必须接入认证、权限校验、审计日志。
- 所有金额字段使用 `BigDecimal` 对应的数据库 `DECIMAL` 类型，不使用浮点数。
- 时间字段必须区分创建时间、更新时间、生效时间、失效时间、审核时间、发布时间。

## 3. 业务状态机

### 3.1 产品状态

| 状态 | 说明 | 可执行动作 | 前置条件 | 操作角色 |
| --- | --- | --- | --- | --- |
| `DRAFT` 草稿 | 产品或新版本可编辑 | 编辑、删除草稿、提交审核 | 未发布或被驳回 | 产品配置员 |
| `IN_REVIEW` 审核中 | 审核流进行中 | 审核通过、驳回、撤回 | 已提交审核 | 审核角色、产品配置员 |
| `REJECTED` 已驳回 | 审核未通过，回到可编辑状态 | 编辑、重新提交 | 存在驳回原因 | 产品配置员 |
| `APPROVED` 已通过 | 审核完成，待上架 | 发布、定时上架 | 发布校验通过 | 产品审批人 |
| `PUBLISHED` 已上架 | 对下游可售或可查询 | 下架、创建新版本 | 已生成发布快照 | 产品审批人 |
| `SUSPENDED` 已下架 | 不可销售，历史可查 | 重新上架、创建新版本 | 未过失效时间且校验通过 | 产品审批人 |
| `EXPIRED` 已失效 | 生命周期结束 | 查看、复制新草稿 | 已过失效时间 | 只读或配置员 |

### 3.2 产品版本状态

| 状态 | 说明 | 可执行动作 |
| --- | --- | --- |
| `DRAFT` | 草稿版本，可编辑 | 保存、提交审核、删除草稿 |
| `IN_REVIEW` | 审核中，不允许普通编辑 | 审核通过、驳回、撤回 |
| `APPROVED` | 审核通过，待发布 | 发布、定时发布 |
| `ACTIVE` | 当前生效版本 | 下架、创建新版本 |
| `INACTIVE` | 已失效历史版本 | 查看、对比、复制 |

状态规则：

- 同一产品同一时间最多一个 `ACTIVE` 版本。
- 发布新版本时必须明确旧版本失效时间。
- 编辑 `ACTIVE` 或 `INACTIVE` 版本必须复制为新的 `DRAFT` 版本。

### 3.3 费率表状态

| 状态 | 说明 | 可执行动作 |
| --- | --- | --- |
| `EDITING` 编辑中 | 可导入、修改、校验 | 保存、校验、提交精算审核 |
| `ACTUARY_REVIEW` 待精算审核 | 精算审核中 | 通过、驳回 |
| `PUBLISHED` 已发布 | 可绑定产品版本 | 废弃、复制 |
| `DISCARDED` 已废弃 | 不可新绑定 | 查看历史 |

### 3.4 审核单状态

| 状态 | 说明 | 可执行动作 |
| --- | --- | --- |
| `PENDING` 待处理 | 当前节点待审核 | 通过、驳回、转交 |
| `APPROVED` 已通过 | 节点完成 | 进入下一节点 |
| `REJECTED` 已驳回 | 流程终止并回草稿 | 产品配置员修改后重新提交 |
| `CANCELLED` 已撤回 | 提交人撤回 | 重新编辑、重新提交 |

### 3.5 供应商合同状态

| 状态 | 说明 | 可执行动作 |
| --- | --- | --- |
| `DRAFT` 草稿 | 合同信息录入中 | 编辑、提交审核 |
| `EFFECTIVE` 生效 | 可关联产品 | 续约、终止 |
| `EXPIRING` 即将到期 | 触发提醒 | 续约、终止 |
| `TERMINATED` 已终止 | 不可新关联产品 | 查看历史 |

## 4. 功能需求

### 4.1 产品中心仪表盘

- 展示产品总数、待审核数、即将停售产品数、即将到期供应商合同数、各状态产品分布。
- 展示分红险数量、分红参数待审核数量、费率表待精算审核数量。
- 提供快捷入口：新建产品、批量导入、待审核列表、费率导入、供应商合同到期列表。
- 展示最近操作日志，包括操作人、操作时间、对象、动作、变更摘要。

验收标准：

- 仪表盘数据按权限过滤。
- 指标点击后可跳转至对应筛选列表。
- 最近操作日志不可展示敏感字段明文。

### 4.2 产品基础信息管理

#### 4.2.1 产品列表

- 支持按产品代码、产品名称、产品类型、传统/分红标识、状态、创建人、日期范围、销售渠道查询。
- 列表字段包括产品代码、产品名称、产品类型、分红标识、当前版本号、状态、最近更新时间、操作人。
- 支持查看详情、编辑草稿、复制产品、创建新版本、提交审核、发布、下架、删除草稿。
- 删除仅允许草稿，已发布业务数据不得物理删除。

#### 4.2.2 创建与编辑产品

- 必填字段：产品代码、产品全称、产品简称、产品类型、传统/分红标识、保险公司、销售渠道。
- 选填字段：宣传语、上市日期、预计停售日期、所属事业部、自定义标签。
- 产品代码全局唯一；发布后不可修改。
- 选择分红型时必须进入分红管理配置。

验收标准：

- 产品代码重复时保存失败，并提示冲突字段。
- 已发布产品不可直接编辑，只能创建新草稿版本。
- 分红型产品缺少分红配置时不能提交发布。

### 4.3 险种与责任配置

#### 4.3.1 险种管理

- 产品版本下可配置多个险种，区分主险、附加险。
- 字段包括险种名称、险种代码、险种类型、保额类型、保费计算方式、排序号。
- 附加险支持排序、启用、停用；已有有效版本引用时不允许物理删除。

#### 4.3.2 责任配置

- 每个险种下可配置多个责任，如身故保险金、全残保险金、满期生存金。
- 字段包括责任名称、责任代码、责任类型、给付层次、理赔类型、保额设定方式、等待期、免责说明、给付条件。
- 支持责任互斥、依赖、与主险保额挂钩公式。

验收标准：

- 每个可发布产品版本至少存在一个主险和一个有效责任。
- 责任代码在同一产品版本内唯一。
- 互斥和依赖关系不能形成冲突或循环。

### 4.4 投保要素与核保规则配置

#### 4.4.1 投保要素字段定义

- 支持模板库维护投保人、被保人、受益人字段集。
- 字段属性包括字段名称、字段编码、类型、是否必填、校验规则、顺序、默认值、字典来源。
- 产品版本可选择模板组，并对单个字段做覆盖配置。
- 支持产品自定义字段，适用于分红领取方式说明等低频扩展场景。

#### 4.4.2 健康告知管理

- 支持健康告知问卷库、问卷版本、题目分类、题型、选项、异常标识、逻辑跳转。
- 产品版本绑定某一问卷版本。
- 异常选项可关联简易核保结论编码，用于前端提示或规则服务调用。
- 支持问卷预览与模拟投保填写。

#### 4.4.3 职业选择与职业限制

- 维护职业分类树、职业代码、风险等级、是否拒保。
- 产品版本可配置白名单、黑名单或按风险等级控制。
- 支持职业加费系数配置，供费率试算引用。

#### 4.4.4 投保规则整体控制

- 配置投保年龄上下限、购买份数限制、保额与年龄/职业联动规则、受益人规则、生效时间规则。
- 复杂核保规则只保存规则编码、规则版本和适用范围。

验收标准：

- 产品发布前必须绑定投保要素模板。
- 健康告知问卷跳转规则模拟时不能出现不可达必答题。
- 职业拒保配置必须在试算和前端校验中可被识别。

### 4.5 投保区间与方案配置

- 管理保险期间、缴费期间基础字典。
- 产品版本下配置保险期间、缴费期间、保额档位、合法搭配关系、默认值。
- 支持保额固定、多档、区间输入，基于年龄、职业设置梯度上限。
- 支持方案与费率表、责任保额公式绑定。
- 提供期间矩阵预览和模拟验证。

验收标准：

- 每个可发布产品版本至少有一个有效投保方案。
- 默认保险期间、缴费期间必须属于合法搭配。
- 模拟验证能明确返回不可用原因。

### 4.6 费率管理

- 费率因子库统一维护年龄、性别、职业等级、保险期间、缴费期间、保额档位等因子。
- 费率表支持 Excel/CSV 批量导入、模板下载、小范围手动录入。
- 导入校验包括必填、类型、值域、重复组合、缺失组合、负费率、异常波动。
- 费率表生命周期为编辑中、待精算审核、已发布、已废弃。
- 同一产品可配置标准、活动折扣、渠道专属等多套费率策略。
- 支持供应商费率、再保费率、佣金比例录入。
- 记录费率变更历史和平均保费变化幅度。

验收标准：

- 10000 行费率数据导入含基础校验不超过 5 秒。
- 导入失败时返回可下载的错误明细。
- 未发布费率表不得绑定已通过或待发布产品版本。

### 4.7 前端联动与验证规则配置

- 后台配置生成前端可渲染的投保页面描述 JSON。
- 规则采用事件-条件-动作模型。
- 事件包括字段值变更、选项选中、组件初始化、模块加载。
- 条件包括等于、不等于、包含、比较、正则、且/或/非组合。
- 动作包括设置 visible、required、readonly、value、options、错误提示、触发试算。
- 支持健康告知跳转、职业与保额联动、责任互斥、分红方式联动。
- 自定义表达式使用受控 DSL，不允许执行任意 JavaScript、SQL、Java 表达式。
- 提供规则模拟器，输入样例数据后实时展示字段状态和校验结果。

验收标准：

- 规则保存前必须通过结构校验。
- 模拟器必须输出命中的规则、动作结果和错误原因。
- 高权限用户才能维护表达式，表达式必须通过白名单校验。

### 4.8 分红管理

- 分红型产品必须配置可选分红方式：现金分红、增额分红、累积生息、混合方式。
- 每种方式可配置规则说明、利率参考、增额计算说明、领取方式限制。
- 精算师维护红利演示低、中、高三档利率假设。
- 红利演示参数与产品版本绑定，修改必须升版审核。
- 配置分红关联责任，如增额后的额外身故保险金。
- 分红参数变更必须经过精算审核和合规审核。

验收标准：

- 分红险缺少分红方式或红利演示参数时，发布校验失败。
- 红利演示利率必须满足低档小于等于中档，中档小于等于高档。
- 分红相关字段必须进入发布快照。

### 4.9 条款与文档关联

- 为产品版本、险种、责任关联条款文件。
- 文件类型包括保险条款、投保须知、费用说明、分红说明、销售材料。
- 支持外部条款预览。
- 新版本创建时提示检查条款版本一致性。

验收标准：

- 发布产品必须至少关联保险条款和投保须知。
- 分红险必须关联分红说明。
- 条款文件 ID、版本、预览地址缺失时不得发布。

### 4.10 产品组合

- 支持创建组合产品 SKU，选择多个已发布寿险单品。
- 支持必选组合、多选一、可选附加险。
- 组合可指定统一费率表，或由单品费率累加。
- 支持组合总保额上限、各险种保额占比限制。

验收标准：

- 组合产品只能引用已发布且未下架产品。
- 组合规则不能出现互斥冲突。
- 组合试算必须返回费用拆分。

### 4.11 审核流程管理

- 审核流采用轻量内置实现。
- 默认流程为产品配置员提交、精算审核、法务合规审核、核保运营审核、产品审批人终审。
- 涉及费率、方案、分红变更时必须进入精算审核。
- 支持并行节点、驳回、审核意见、附件引用、撤回、超时提醒。
- 提供待我审核列表、已办列表、审核详情、审核历史。

验收标准：

- 驳回必须填写原因。
- 审核中产品不允许普通编辑。
- 审核记录按时间顺序完整展示，不可删除。

### 4.12 版本与生命周期管理

- 编辑已发布产品自动生成新草稿版本。
- 支持历史版本详情、版本差异对比、基于历史版本生成新草稿。
- 生命周期为草稿、审核中、已驳回、已通过、已上架、已下架、已失效。
- 上架条件包括基础信息、责任、费率、条款、投保要素、方案、规则、分红参数完整且有效。
- 支持定时上架、定时下架任务。

验收标准：

- 同一产品不能存在多个当前生效版本。
- 发布失败不得污染当前生效版本。
- 下架不影响历史保单和历史快照查询。

### 4.13 产品查询与报表

- 支持产品目录导出、费率摘要导出、责任摘要导出。
- 支持产品血缘关系图，展示产品、险种、责任、费率表、条款、供应商、分红配置的关联。
- 支持操作日志审计查询，不允许删除审计日志。
- 导出必须受权限控制并限制数据量。

验收标准：

- 导出任务有权限校验、数据量上限和操作日志。
- 血缘图可定位到具体版本。
- 审计日志支持按操作人、对象、动作、时间查询。

### 4.14 供应商管理

- 维护供应商基础档案：名称、代码、类型、有效期、合作状态、评级。
- 维护证照和合同，到期提醒。
- 再保供应商可关联产品，录入分保比例、分出方式、再保费率。
- 服务供应商可关联增值服务，配置结算方式。
- 渠道供应商可关联销售渠道，配置佣金比例。
- 维护供应商能力与准入，产品配置时按寿险、服务类别、合作状态自动过滤。
- 支持供应商费率与对账数据录入。

验收标准：

- 已终止或过期供应商不能新关联产品。
- 供应商合同到期前可配置提醒。
- 产品发布快照包含有效供应商关联。

### 4.15 系统基础数据管理

- 产品类型字典：定期寿险、终身寿险、两全保险，区分传统/分红。
- 保险期间、缴费期间、销售渠道、责任类型、分红方式、供应商类型、费率因子字典。
- 审批流模板配置，支持按产品类型启用节点。
- 投保要素字段模板库、健康告知分类、职业分类树。

验收标准：

- 已被发布产品引用的字典值不得物理删除。
- 字典变更必须记录审计日志。
- 字典停用后不能被新配置引用。

## 5. 发布快照与数据一致性

### 5.1 发布快照内容

产品版本发布成功后必须生成不可变发布快照，至少包含：

- 产品基础信息、产品版本、生命周期状态。
- 险种、责任、责任互斥和依赖关系。
- 投保方案、保险期间、缴费期间、保额档位、合法组合矩阵。
- 费率策略、费率表索引、费率因子定义。
- 投保要素字段、字段覆盖配置、健康告知问卷版本、职业限制。
- 前端联动规则集、验证规则、前端渲染页面描述 JSON。
- 条款文件索引、文件版本、预览地址。
- 分红方式、红利演示参数、分红关联责任。
- 供应商关联、再保比例、佣金比例、服务关联。

### 5.2 一致性规则

- 开放查询接口只读取已发布快照或快照缓存。
- 发布快照不可变，版本变更必须生成新快照。
- 产品下架只改变销售状态，不删除快照。
- 发布事务必须保证数据库中产品版本状态与快照记录一致。
- Redis 刷新失败不回滚数据库事务，但必须记录告警并进入补偿任务。

## 6. 非功能需求

### 6.1 性能

- 产品列表加载不超过 2 秒，支持 2000 条以内常规筛选。
- 开放产品详情查询 P95 不超过 100ms，优先读取 Redis。
- 开放查询接口支持 200 TPS。
- 费率表导入 10000 行含基础校验不超过 5 秒。

### 6.2 可用性

- 核心服务可用性不低于 99.9%。
- 计划内停机每月不超过 1 小时。
- 数据库至少每小时备份一次。
- 发布失败可快速回滚到上一有效快照。

### 6.3 安全

- 后台接口必须接入统一认证和权限校验。
- 敏感操作支持二次身份验证。
- 管理端权限控制到菜单和按钮。
- 日志禁止打印身份证号、手机号、银行卡号等敏感信息明文。
- 动态表达式必须白名单化，禁止任意脚本执行。

### 6.4 可扩展性

- 产品模型支持受控动态字段。
- 费率因子、投保要素、健康告知、审批流、联动规则支持配置化。
- 复杂规则通过外部规则服务引用，产品中心保存规则编码和版本。

### 6.5 可观测性

- 接口日志包含 traceId。
- 发布、下架、导入、导出、审核、规则变更记录业务日志。
- 接入 Prometheus 监控接口耗时、错误率、缓存命中率、导入任务耗时。
- 错误日志接入 ELK。

## 7. 数据需求

### 7.1 核心实体关系

- 产品 `Product` 1 对多 产品版本 `ProductVersion`。
- 产品版本 1 对多 险种 `RiskItem`、投保方案 `InsurancePlan`。
- 产品版本 1 对 1 或 0 对 1 分红配置 `DividendConfig`。
- 险种 1 对多 责任 `CoverageLiability`。
- 产品版本关联费率策略 `RateStrategy`，费率策略关联费率表 `RateTable`。
- 产品版本多对多关联条款文件 `ClauseDocument`。
- 产品版本关联投保要素模板、健康告知问卷版本、联动规则集。
- 产品可组成组合产品 `ProductCombo`。
- 供应商与产品多对多关联。
- 审核记录关联产品版本。
- 发布快照关联产品版本。

### 7.2 基础数据字典

| 字典类型 | 示例值 |
| --- | --- |
| 产品类型 | 定期寿险、终身寿险、两全保险 |
| 产品属性 | 传统、分红 |
| 分红方式 | 现金分红、增额分红、累积生息、混合 |
| 责任类型 | 身故、全残、满期生存 |
| 保险期间 | 20 年、至 70 周岁、终身 |
| 缴费期间 | 趸交、10 年交、20 年交 |
| 产品状态 | 草稿、审核中、已通过、已上架、已下架 |
| 销售渠道 | APP、H5、小程序、线下经代 |
| 供应商类型 | 再保、服务、渠道 |
| 费率因子 | 年龄、性别、职业等级、保险期间、缴费期间 |

## 8. 接口需求

### 8.1 开放接口

- 获取最新生效产品定义：返回产品基础信息、险种责任、投保要素、健康告知、职业规则、投保方案、联动规则 JSON、费率索引、条款索引；分红险额外返回分红方式与演示参数。
- 获取指定版本产品定义：按产品代码和版本号读取历史快照。
- 批量产品定义同步：支持按更新时间增量拉取。
- 产品上下架状态变更通知：通过消息队列推送。
- 费率试算接口：传入因子、方案、保额、职业、渠道、分红方式，返回保费和分红演示值。

### 8.2 内部接口

- 条款预览服务。
- 通知服务：站内信、邮件、企业微信。
- 供应商服务：验证供应商有效性。
- 精算引擎：获取红利演示参数或复杂试算结果。
- 规则引擎：验证核保规则引用有效性。

## 9. 验收标准汇总

- 产品创建、编辑、提交审核、驳回、审批、发布、下架全流程可闭环。
- 分红险缺少分红方式、红利演示参数、分红说明时不可发布。
- 发布校验失败必须返回结构化失败原因和对应模块。
- 已发布版本不可原地修改，创建新版本后旧版本仍可查询。
- 开放接口读取发布快照，结果不受草稿修改影响。
- 费率导入支持错误明细下载。
- 联动规则模拟器可展示命中规则和执行结果。
- 所有写操作都有审计日志。
- 导出、发布、下架、费率审核等敏感操作有权限控制。

## 10. 风险与约束

- 前端联动规则复杂度高，首版必须限制为结构化 ECA 和受控表达式 DSL。
- 费率维度过多时，明细表规模可能快速增长，需要基于产品规模选择索引、分表、预计算或缓存策略。
- 审核流首版采用轻量内置实现，若后续出现复杂会签、动态流程图、跨系统流程协同，再评估 BPMN 引擎或 OA 对接。
- 分红演示结果必须明确为演示值，真实红利分配以核心系统或精算引擎为准。
- 供应商合同、条款文件和外部规则引用依赖外部系统可用性，产品中心需保存引用快照和校验结果。
